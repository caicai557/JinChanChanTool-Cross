# 双端版优化补充优先级计划

## 目标
在不破坏现有热键语义和配置兼容的前提下，把双端版补齐到可发布主线。

## M1（P0）：先跑通建议链路关键缺口
- 补 `IGameStateReader` 的备战席输入能力。
- 建立建议快照统一输出路径。
- 覆盖层从空实现改为平台实现注入。

验收：
- 建议快照可稳定输出。
- 覆盖层实现可被调用且可清理。

## M2（P0）：阵容数据源升级
- 阵容模板从硬编码提升到本地数据源加载。
- 支持英雄名/羁绊/首字母检索保持不变。
- 保留内置模板兜底。

验收：
- 数据文件损坏时仍可回退默认模板。
- 搜索与推荐测试通过。

## M3（P1）：可视化与交互完善
- 主界面增加可读性更强的建议展示结构。
- 建议摘要、优先级、兜底原因可见。

验收：
- 用户可在一屏内看到阵容、资源、装备、符文建议。

## M4（P0）：稳定性与发布
- 统一指标输出与异常日志格式。
- 固化 CI 构建与双端产物打包。
- 完成回归清单并形成发布说明。

验收：
- Core/Platform/E2E 测试全绿。
- Windows/macOS 构建流水线可重复。

## M5（P0）：删除旧 Windows 稳定版
- 删除旧工程目录、旧解决方案和旧引用。
- README 与文档切换为双端主线。
- 保留 `windows-stable-final` 标签作为回滚点。

验收：
- 仓库只保留双端工程。
- 发布与测试链路不再依赖旧版代码。
